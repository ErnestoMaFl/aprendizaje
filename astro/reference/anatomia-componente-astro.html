Anatomía de un componente .astro

La chuleta de una sola página. Vuelve aquí cuando dudes de la estructura.

Las dos zonas de todo archivo .astro

---              // 1) SCRIPT DEL COMPONENTE (frontmatter)
import Tarjeta from '../components/Tarjeta.astro';
const titulo = 'Servicios';
const { destacado = false } = Astro.props;   // props recibidas
---
<!-- 2) PLANTILLA (HTML + expresiones) -->
<h1>{titulo}</h1>
{destacado && <span>★</span>}
<Tarjeta />

① Script (entre ---)

  • Es JavaScript/TypeScript normal.
  • Corre en el servidor / en el build, nunca en el navegador.
  • Todo lo que pongas aquí se elimina del HTML final que recibe el usuario.
  • Sirve para: import, definir variables, obtener datos (fetch, await), leer Astro.props.

② Plantilla (debajo del ---)

  • Parece HTML, con extras tipo JSX.
  • Expresiones con llaves: {variable}.
  • Listas con {items.map(i => <li>{i}</li>)}.
  • Condicionales con {cond && <p>…</p>}.
  • Puedes usar otros componentes como etiquetas: <Tarjeta />.

Piezas clave del vocabulario

TérminoQué es
--- (code fence)Las dos líneas de tres guiones que delimitan el script del componente.
FrontmatterOtro nombre para el script del componente (por analogía con Markdown).
Astro.propsObjeto con los datos que el componente recibe desde quien lo usa.
<slot />Hueco donde se inyecta el contenido hijo pasado al componente.
Cero JS por defectoEl HTML enviado no lleva JavaScript salvo que lo pidas explícitamente (islas).

La idea que lo explica todo

Un componente .astro es una plantilla que se ejecuta una vez en el servidor y produce HTML plano. Por eso el sitio es rápido: el navegador recibe HTML terminado, no un programa que tenga que ejecutar.
