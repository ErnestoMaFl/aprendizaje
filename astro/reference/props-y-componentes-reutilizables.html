Props y componentes reutilizables

La chuleta para escribir un componente una vez y usarlo mil veces.

El circuito completo de una prop

// src/components/Tarjeta.astro — el componente DEFINE qué recibe
---
const { icono = '🥖', titulo, texto } = Astro.props;
---
<article>
  <span>{icono}</span> <h3>{titulo}</h3> <p>{texto}</p>
</article>

// src/pages/index.astro — quien lo USA le pasa los datos como atributos
---
import Tarjeta from '../components/Tarjeta.astro';
---
<Tarjeta titulo="Pan artesanal" texto="Horneado cada mañana." />

Pasar props (quien usa)

  • Texto fijo, como atributo HTML: titulo="Pan artesanal".
  • Cualquier valor JS, entre llaves: precio={4.5}, activo={true}, items={lista}.
  • El componente se importa en el frontmatter y se usa como etiqueta: <Tarjeta … />.
  • Sin contenido hijo, ciérralo en la propia etiqueta: />.

Recibir props (el componente)

  • Todas llegan en el objeto Astro.props.
  • Lo habitual es desestructurar: const { titulo, texto } = Astro.props;
  • Valor por defecto si no la pasan: const { icono = '🥖' } = Astro.props;
  • En la plantilla se usan con llaves: {titulo} — también dentro de atributos: href={url}.

Reglas y errores frecuentes

ReglaDetalle
Dónde vivenPor convención, en src/components/. Las páginas, en src/pages/.
Nombre en mayúsculaEl componente se usa como <Tarjeta /> — la mayúscula inicial lo distingue de una etiqueta HTML.
Solo lecturaLas props se leen una vez en el build. No hay estado ni re-render: si el dato cambia, se reconstruye el sitio.
Astro 7 es estrictoEl compilador ya no corrige HTML mal formado: una etiqueta sin cerrar o un atributo a medias detienen el build con error (antes se autocorregían en silencio).
Tipado (opcional)La documentación tipa las props con TypeScript: interface Props { titulo: string; icono?: string }. Da autocompletado y avisa si olvidas una prop obligatoria.

La idea que lo explica todo

Un componente con props es una plantilla con huecos: el HTML común se escribe una sola vez y las props rellenan lo que cambia. Cuando el cliente pida un cambio de diseño, lo tocas en un archivo y se aplica en todo el sitio.
