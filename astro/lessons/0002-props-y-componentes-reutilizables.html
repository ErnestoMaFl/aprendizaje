Props: escribe la tarjeta una vez, úsala mil veces

La victoria de hoy: convertir HTML repetido en un componente con huecos, y rellenar esos huecos con props — la pieza central de toda web de cliente mantenible.

🎯 Al terminar podrás: crear un componente en src/components/, pasarle datos como atributos y leerlos con Astro.props (con valores por defecto), y explicar por qué eso te ahorra horas cuando el cliente pida cambios.

Primero, un problema (intenta antes de leer)

La Panadería Aurora quiere una sección de «Servicios» con tres tarjetas. Tu primer impulso podría ser este HTML en la página de inicio:

<article class="tarjeta">
  <span>🥖</span> <h3>Pan artesanal</h3> <p>Horneado cada mañana.</p>
</article>
<article class="tarjeta">
  <span>🎂</span> <h3>Pasteles por encargo</h3> <p>Para bodas y eventos.</p>
</article>
<article class="tarjeta">
  <span>☕</span> <h3>Café de especialidad</h3> <p>De tostado local.</p>
</article>

Tu predicción: el cliente llama y pide añadir un precio a todas las tarjetas (y la semana que viene habrá tarjetas también en la página «Sobre nosotros»). ¿Qué problema le ves a este enfoque, y — con lo que viste de Astro.props en la lección 1 más lo que sepas de React — cómo intentarías resolverlo? El problema es la repetición: la estructura de la tarjeta vive en tres sitios (pronto más). Cada cambio de diseño hay que aplicarlo N veces, y tarde o temprano una tarjeta se queda distinta de las demás.

La solución en Astro: escribir la estructura una sola vez en un componente (Tarjeta.astro) dejando huecos para lo que cambia (icono, título, texto), y rellenar esos huecos con props al usarlo:

<Tarjeta icono="🥖" titulo="Pan artesanal" texto="Horneado cada mañana." />
<Tarjeta icono="🎂" titulo="Pasteles por encargo" texto="Para bodas y eventos." />
<Tarjeta icono="☕" titulo="Café de especialidad" texto="De tostado local." />

¿Precio nuevo en todas? Se añade una vez en Tarjeta.astro y listo. Si vienes de React esto te suena: es la misma idea de props — la diferencia es dónde corre, y eso lo vemos ahora.

El circuito de una prop: definir → pasar → usar

Contexto que necesitas primero (2 carpetas, nada más). Aún no hemos montado ningún proyecto — eso llega en la próxima lección —, pero para leer los ejemplos de hoy te bastan dos convenciones de todo proyecto Astro:1

  • src/pages/ — las páginas del sitio: cada archivo aquí se convierte en una URL (index.astro es la portada).
  • src/components/ — las piezas reutilizables que las páginas usan (nuestra Tarjeta.astro).

Consecuencia práctica: como la página vive en src/pages/ y el componente en src/components/, la página lo importa con una ruta relativa entre carpetas hermanas: '../components/Tarjeta.astro' (sube un nivel, entra en components), siempre entre comillas por ser un import de JavaScript. Por ahora léelo como contexto; lo teclearás de verdad cuando montemos el proyecto.

Este es Tarjeta.astro completo — fíjate en que son las dos zonas que ya conoces (script + plantilla):2

// src/components/Tarjeta.astro
---
const { icono = '🍞', titulo, texto } = Astro.props;  // ① DEFINE sus huecos
---
<article class="tarjeta">
  <span>{icono}</span>
  <h3>{titulo}</h3>
  <p>{texto}</p>
</article>

Y así lo usa la página de inicio:

// src/pages/index.astro
---
import Tarjeta from '../components/Tarjeta.astro';   // ② lo IMPORTA
---
<section class="servicios">
  <Tarjeta icono="🥖" titulo="Pan artesanal" texto="Horneado cada mañana." />
  <Tarjeta titulo="Café de especialidad" texto="De tostado local." />
</section>                                // ③ le PASA los datos como atributos

Tres piezas que encajan:2

  • Definir. Todas las props llegan al componente en el objeto Astro.props. Lo habitual es desestructurarlas: const { titulo, texto } = Astro.props;. Un valor por defecto (icono = '🍞') cubre el caso en que no la pasen — mira la segunda tarjeta: no lleva icono, y saldrá 🍞.
  • Pasar. Quien usa el componente le pasa los datos como atributos HTML. Texto fijo entre comillas (titulo="Pan artesanal"); cualquier valor de JavaScript entre llaves (precio={4.5}, destacada={true}).
  • Usar. En la plantilla, las llaves insertan la prop donde toque: {titulo} — y también dentro de atributos: href={url}.

Contraste con React. El modelo mental es el mismo (props hacia abajo, componente en mayúscula como <Tarjeta />), pero aquí las props se leen una vez en el build, en el servidor. No hay estado ni re-render: el resultado es HTML terminado. Si el dato cambia, se reconstruye el sitio — para la web de un cliente (contenido que cambia poco), eso es una ventaja, no una limitación.

Novedad de Astro 7 que te va a pasar. El compilador nuevo (reescrito en Rust) es estricto como JSX: una etiqueta sin cerrar o un atributo a medias ya no se autocorrigen en silencio — detienen el build con un error que señala la línea.3 Al principio molesta; en la web de un cliente es un regalo: el HTML roto se detecta en tu máquina, no en el navegador del visitante. Ojo con los componentes sin contenido: ciérralos en la propia etiqueta, <Tarjeta … />.

En la documentación verás las props tipadas con TypeScript (interface Props { titulo: string }). Es el estándar de los proyectos serios porque avisa si olvidas una prop obligatoria — lo incorporaremos cuando montemos el proyecto real; hoy no lo necesitas.

Practícalo (de más guiado a menos)

Paso 1 · Completa el componente

La panadería quiere una cita de un cliente satisfecho. La plantilla ya está hecha; te falta el frontmatter. Escribe la línea que extrae nombre y frase, y estrellas con '⭐⭐⭐⭐⭐' por defecto:

// src/components/Testimonio.astro
---
// … tu línea aquí …
---
<blockquote>
  <p>«{frase}»</p>
  <footer>— {nombre} <span>{estrellas}</span></footer>
</blockquote>
Comparar con la solución del paso 1 
const { nombre, frase, estrellas = '⭐⭐⭐⭐⭐' } = Astro.props;

Claves: se desestructura desde Astro.props, y el valor por defecto va con = dentro de las llaves de desestructuración (es JavaScript normal, el mismo que usarías en cualquier función).

Paso 2 · Úsalo desde la página

Ahora escribe las dos líneas que faltan en src/pages/index.astro para mostrar el testimonio de Marta («El mejor pan del barrio»): la de importar y la de usar el componente. Tienes el patrón exacto en el ejemplo de index.astro de arriba — la ruta funciona igual aquí.

Comparar con la solución del paso 2 
---
import Testimonio from '../components/Testimonio.astro';
---
<Testimonio nombre="Marta" frase="El mejor pan del barrio" />

El import siempre va en el frontmatter (entre los ---). Las props de texto van como atributos con comillas, y como no le pasamos estrellas, saldrá el valor por defecto. Nota Astro 7: sin el /> de cierre, el build fallaría con error.

Paso 3 · Tú solo: el botón del sitio

Sin mirar los pasos anteriores: escribe completo un componente Boton.astro que reciba texto, url y tipo (por defecto 'primario'), y pinte un enlace <a> cuya class sea el tipo y cuyo href sea la url. Añade también una línea de ejemplo de cómo lo usaría la página.

Comparar con la solución del paso 3 
// src/components/Boton.astro
---
const { texto, url, tipo = 'primario' } = Astro.props;
---
<a class={tipo} href={url}>{texto}</a>

// y en la página, por ejemplo:
<Boton texto="Haz tu pedido" url="/contacto" />

Fíjate: class={tipo} y href={url} — las llaves funcionan también dentro de atributos. Si tu versión difiere en nombres pero el circuito definir→pasar→usar está bien, cuenta como correcta.
