Lección 02 · JavaScript del 80%

Funciones: leer la flecha =>

En Python solo hay una forma de escribir una función: def. En JavaScript la misma función se puede escribir de cinco maneras que parecen distintas y hacen lo mismo — y de una sexta que parece igual y devuelve basura. Eso es lo que hace ilegible el código ajeno. Al terminar sabrás desenredar cualquier flecha hasta su forma larga, y detectar la que está rota.

Empieza fallando: predice antes de leer nada

Cinco maneras de escribir «dame el doble de un número». Todas se guardan en un const, como aprendiste en la lección anterior — sí: en JavaScript una función es un valor que se guarda en una variable. 

const doble1 = function (n) { return n * 2; };
const doble2 = (n) => { return n * 2; };
const doble3 = (n) => n * 2;
const doble4 = n => n * 2;
const doble5 = n => { n * 2; };

console.log(doble1(5), doble2(5), doble3(5), doble4(5), doble5(5));

¿Qué imprime esa última línea? Escribe los cinco valores. Uno de los cinco no vale 10 — y ese es el que te va a costar horas de depuración algún día si no lo cazas hoy.

La escalera de contracción

Las cuatro primeras son la misma función, cada una un poco más contraída que la anterior. No son cuatro cosas que aprender: son cuatro peldaños de una sola escalera. Léela de arriba abajo una vez y ya sabrás subir y bajar por ella.

0 
const doble = function (n) { return n * 2; };

Punto de partida: la palabra function, sin nombre. Es lo que se escribía antes de 2015 y lo que verás en código antiguo.

1 
const doble = (n) => { return n * 2; };

Quitas function y pones => entre los paréntesis y las llaves. Mismo significado, cinco letras menos.

2 
const doble = (n) => n * 2;

El peldaño importante. Si el cuerpo es una sola expresión, puedes quitar las llaves — y al quitarlas return se vuelve obligatorio quitarlo: se devuelve solo. Es el return implícito.

3 
const doble = n => n * 2;

Con exactamente un parámetro, los paréntesis también sobran. Con cero o con dos, vuelven a ser obligatorios: () => 42, (a, b) => a + b.

La regla que lo resume todo

Llaves { } después de la flecha ⇒ es un bloque de instrucciones, y necesita return. Sin llaves ⇒ es una expresión, y se devuelve sola. Esa única frase explica los cinco casos del pretest.

El que devuelve basura

doble5 tiene llaves, así que su contenido es un bloque de instrucciones. Dentro hay una instrucción, n * 2, que calcula 10… y lo tira a la basura, porque nadie dijo return. Una función que llega al final sin return devuelve undefined: el valor que JavaScript usa para «aquí no hay nada».

Y no da error. Ese es el problema: te devuelve undefined en silencio y el fallo aparece tres funciones más allá. En Python el equivalente sería un def sin return, que devuelve None — misma idea, pero allí no te pasa por accidente al contraer una línea.

El caso que confunde a todo el mundo: devolver un objeto

Si el valor que quieres devolver es un objeto, tienes un choque: las llaves del objeto parecen las llaves de un bloque. JavaScript siempre elige bloque. La solución es envolver el objeto en paréntesis: 

const crearCliente = nombre => { nombre: nombre };    // ✖ devuelve undefined
const crearCliente = nombre => ({ nombre: nombre });  // ✔ devuelve el objeto

Ese => ({ … }) aparece constantemente en código real. Ahora ya sabes que los paréntesis no son adorno: son los que dicen «esto es un valor, no un bloque».

La otra forma: function con nombre

Además de guardarse en una variable, una función se puede declarar con nombre propio. Es lo más parecido al def de Python y lo verás por todas partes: 

function calcularIva(base) {
  return base * 0.21;
}
FormaCuándo la verás
const f = () => … núcleo La opción por defecto hoy, sobre todo para funciones cortas y para pasarlas como argumento.
function f() { … } núcleo Muy viva para funciones «principales» con nombre. Escríbela sin complejos cuando la función es grande.
const f = function () { … } solo leer La forma pre-2015. Reconócela: es una flecha sin contraer.

Aviso honesto, para que no te lo encuentres de golpe

Las flechas y function no son intercambiables al 100%: se comportan distinto con una palabra llamada this, y esa diferencia tiene lección propia más adelante. Mientras no aparezca this en el código —que es la mayoría del tiempo— puedes tratarlas como equivalentes.

Parámetros por defecto

Igual que en Python, y se lee igual:

const saludar = (nombre, saludo = "Hola") => saludo + ", " + nombre;

saludar("Ana")              // "Hola, Ana"
saludar("Ana", "Buenas")   // "Buenas, Ana"

Detalle que usarás mucho: el + entre textos los pega. "Hola" + ", " + "Ana" da "Hola, Ana". (Hay una forma más cómoda de hacer esto, las plantillas; tendrá su lección.)

Para qué tanta contracción: pasar funciones como argumento

Aquí está la razón de que la flecha exista. En JavaScript una función es un valor, así que se puede pasar a otra función igual que se pasa un número. Una función que recibes como argumento se llama callback. 

const aplicarDosVeces = (f, valor) => f(f(valor));

aplicarDosVeces(n => n * 2, 5)   // 20 — dobla, y vuelve a doblar

Fíjate en f(f(valor)): f a secas es la función; f(…) con paréntesis es llamarla. Esa distinción es la que más se confunde al empezar, y es exactamente igual que en Python.

Escrita larga, la llamada de arriba ocuparía tres líneas para nada. Por eso ganó la flecha: cuando la función es un argumento de paso, contraerla hace el código más legible, no menos. Esta misma forma es la que verás en addEventListener("click", () => …) o en lista.map(x => …), que llegarán en sus lecciones.

Compruébalo — recuerda sin releer

Pregunta 1
¿Qué devuelve precio(10)? 

const precio = base => { base * 1.21; };

Correcto. Hay llaves, así que es un bloque: calcula 12.1 y lo descarta porque falta return. La función termina sin devolver nada → undefined. Y lo peor es que no protesta: el fallo aparecerá más adelante.
Aplica la regla: llaves ⇒ bloque ⇒ hace falta return. Aquí no lo hay, así que el 12.1 se calcula y se tira. La función devuelve undefined, y sin dar ningún error. Para que devolviera 12.1: base => base * 1.21, sin llaves.

Pregunta 2
¿Cuál de estas tres flechas no es válida?

Correcto. Los paréntesis solo se pueden omitir con exactamente un parámetro. Con dos son obligatorios: (a, b) => a + b. Con cero también: () => 42.
Las otras dos son correctas: sin parámetros los paréntesis vacíos son obligatorios (() => 42), y con un solo parámetro se pueden quitar (a => a + 1). La que falla es la de dos parámetros sin paréntesis: ahí sí son obligatorios.

Pregunta 3 · escribe la respuesta
¿Qué valor devuelve en JavaScript una función que termina sin return? (una palabra, tal como la escribiría JavaScript)

Correcto: undefined. Es el valor de «aquí no hay nada» — el equivalente del None que devuelve un def sin return en Python. Verlo aparecer en consola casi siempre significa «se me olvidó un return».
La respuesta es undefined. No es null (ese lo escribe un humano a propósito para decir «vacío intencionado»), ni None (eso es Python), ni un error: JavaScript devuelve undefined en silencio.

Pregunta 4
En aplicarDosVeces(doble, 5), ¿por qué se escribe doble y no doble()?

Correcto. doble es la función como valor; doble() la ejecuta ya y pasaría lo que devuelva. Con los paréntesis estarías entregando un número, no una herramienta. Idéntico a Python.
Los paréntesis nunca son opcionales: son el gesto de llamar. doble = la función misma; doble() = ejecútala ahora y quédate el resultado. Como aquí queremos que aplicarDosVeces la llame ella misma (dos veces), hay que entregarla sin llamar.

Pregunta 5 · repaso
(de la Lección 01) ¿Qué ocurre con este código? 

const doble = n => n * 2;
doble = n => n * 3;

Correcto. Que el valor sea una función no cambia nada: doble es una etiqueta pegada con const, y la segunda línea intenta despegarla → TypeError. Para poder cambiar la función habría que declararla con let.
Recuerda la etiqueta y la caja: la segunda línea despega la etiqueta doble para pegarla a otra función. Eso es reasignar, y const lo prohíbe → TypeError: Assignment to constant variable. Da igual que dentro haya una función y no un número.

Antes de la misión: en qué orden se resuelve una cuenta

Añadido el 2026-07-21, porque lo usé sin haberlo dicho nunca. JavaScript sigue las reglas de siempre: * y / se resuelven antes que + y -, y los paréntesis mandan sobre todo. Idéntico a Python y a las matemáticas del colegio. 

base + base * 0.21     // con base=1000 → 1000 + 210 = 1210
(base + base) * 0.21   // con base=1000 → 2000 × 0.21 = 420

Es fácil leer la primera como si fuera la segunda cuando vas rápido. Al leer código ajeno, si una cuenta mezcla + y *, párate un segundo y resuelve primero la multiplicación.

Tu misión de lectura

Código real de una web de cliente: calcula el presupuesto de un proyecto. Léelo sin ejecutarlo y responde las cuatro preguntas. 

const IVA = 0.21;

const conIva = base => base + base * IVA;
const descuento = (base, porcentaje = 10) => base - base * (porcentaje / 100);
const aplicar = (f, base) => f(base);
const resumen = total => { total: total };

console.log(conIva(1000));
console.log(descuento(1000));
console.log(aplicar(conIva, 2000));
console.log(resumen(500));

a) ¿Qué imprime cada una de las cuatro líneas?   b) ¿Por qué descuento(1000) funciona con un solo argumento?   c) En aplicar(conIva, 2000), ¿qué pasaría si estuviera escrito aplicar(conIva(), 2000)?   d) resumen está roto. Arréglalo.

Pista (si la necesitas)
Para cada flecha, hazte una sola pregunta: ¿hay llaves después de la flecha? Si las hay, busca el return; si no está, ya sabes lo que devuelve. Y para d), recuerda el choque entre las llaves de un objeto y las de un bloque.
Ver la solución (solo después de escribir la tuya)

a) 1210 · 900 · 2420 · undefined.
b) Porque porcentaje tiene valor por defecto (= 10): si no se lo pasas, vale 10.
c) conIva() la llama ya, sin argumento, así que base sería undefined y devolvería NaN («no es un número»). Luego aplicar intentaría llamar a ese NaN como si fuera una función → TypeError: f is not a function. Pasar conIva sin paréntesis es justo lo que evita esto.
d) const resumen = total => ({ total: total }); — los paréntesis convierten las llaves en un objeto en vez de un bloque.

Si sacaste la d) sin la pista, ya lees flechas mejor que buena parte de la gente que las escribe a diario.
