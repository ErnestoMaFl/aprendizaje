Lección 02 · Leer CSS

Selectores: ¿a qué apunta?

En la Lección 01 viste que toda regla es selector { propiedad: valor; }. Ya sabes leer el bloque. Hoy conquistas el selector: la parte que decide qué elementos de la página reciben esos estilos. Es la habilidad #1 para leer CSS ajeno.

Los tres selectores que verás el 90% de las veces

Todo se reduce a mirar el primer símbolo del selector:

Se ve asíSímboloApunta a…Se lee
h1 { } (nada) Un tipo de etiqueta HTML: todos los <h1>. «todos los h1»
.card { } . Una clase: todo elemento con class="card". «los que tienen clase card»
#menu { } # Un id: el único elemento con id="menu". «el que tiene id menu»

La pista está en el primer carácter

. punto → clase (se puede repetir en muchos elementos).   # almohadilla → id (debería ser único en la página).   Sin símbolo → el nombre de una etiqueta HTML.

Recuerda el punto misterioso de la lección pasada: .card. Ya lo sabes leer — es «los elementos con clase card». Enlaza con el HTML así:

/* HTML */  <div class="card"> … </div>
/* CSS  */  .card { background: white; }

Combinar selectores: leer una "cadena"

Los selectores se pegan unos a otros. Dos símbolos hacen casi todo el trabajo pesado en código real: el espacio y la coma.

Espacio = "dentro de"

Un espacio entre dos selectores significa "el segundo, pero solo cuando está dentro del primero":

nav a { … }   /* los enlaces (a) que están dentro de un nav */

Léelo de derecha a izquierda: «los a… que estén dentro de un nav». Los a del resto de la página no se tocan.

Coma = "y también"

Una coma aplica la misma regla a varios selectores independientes:

h1, h2, .titulo { color: navy; }
/* los h1, Y los h2, Y los de clase titulo: todos navy */

También te encontrarás (reconócelos, no los memorices)

.card>p — > es "hijo directo" (un nivel, no anidado hondo).   a:hover — : es una pseudo-clase: un estado ("cuando el ratón está encima").   input[type="text"] — [ ] filtra por un atributo.

Compruébalo — recuerda sin releer

Pregunta 1
El selector .boton { } apunta a…

Correcto. El punto . significa clase: cualquier elemento con class="boton".
El primer símbolo manda: . = clase. Apunta a los elementos con class="boton". El # sería id; sin símbolo sería una etiqueta.

Pregunta 2
¿Qué selecciona footer a (con un espacio en medio)?

Correcto. Espacio = "dentro de". Léelo al revés: los a que estén dentro de un footer.
El espacio significa "dentro de", leído de derecha a izquierda: los a dentro de un footer. La coma sería "y también"; sin coma no es eso.

Pregunta 3
En h1, p { margin: 0; }, la coma significa que el margin: 0 se aplica a…

Correcto. La coma une selectores independientes: mismo estilo para h1 y para p.
La coma es "y también": aplica la misma regla a los h1 y a los p por separado. El "dentro de" sería un espacio sin coma.

Pregunta 4 · repaso
En .card { padding: 16px; }, ¿cuál es la propiedad? (de la Lección 01)

Correcto. padding es la propiedad (el "qué"); 16px su valor; .card el selector.
Repasa la Lección 01: la propiedad va antes de los : → aquí es padding. 16px es el valor, .card el selector.

Tu misión de lectura (45 segundos)

Traduce este selector real en voz alta, de derecha a izquierda, antes de seguir:

.menu li a { color: white; }

Pista: son tres piezas encadenadas con espacios. ¿A qué apunta exactamente?   (Respuesta: los enlaces a, que estén dentro de un li, que a su vez esté dentro de algo con clase menu.)
