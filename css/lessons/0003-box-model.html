Lección 03 · Leer CSS

El modelo de caja

Ya sabes leer a qué apunta una regla (Lección 02) y qué le cambia (Lección 01). Hoy respondes la pregunta que confunde a todo el mundo al leer un layout: ¿por qué este elemento ocupa el espacio que ocupa? La respuesta está en cuatro capas.

Antes de leer: arriésgate (aunque no lo sepas)

Un elemento tiene padding: 20px; y también margin: 20px;. Uno de los dos empuja hacia dentro (separa el contenido de su propio borde) y el otro hacia fuera (lo separa de los elementos vecinos). ¿Cuál crees que es cuál? Apuesta mentalmente — luego lo compruebas.

Todo elemento es una caja de 4 capas

El navegador dibuja cada elemento como una caja con cuatro capas, de dentro hacia fuera. Leerlas en ese orden es la clave:

margin · espacio EXTERIOR
border · el borde
padding · espacio INTERIOR
content tu texto / imagen
CapaQué esRegla para leerla
contentEl contenido real: texto, imagen.Lo que ves «lleno».
paddingRelleno dentro del borde.Aleja el contenido de su propio borde.
borderLa línea del borde.La frontera visible (o invisible) de la caja.
marginEspacio fuera del borde.Separa esta caja de las de al lado.

La confusión #1 al leer CSS — resuélvela ya

padding = dentro (empuja el contenido lejos del borde).   margin = fuera (empuja a los vecinos).   Truco: pad = acolchado interior; margen = el margen de una hoja, el aire de alrededor. ¿Acertaste tu apuesta de arriba?

Leer los atajos: margin: 10px 20px;

Casi nunca verás los cuatro lados escritos uno por uno. Verás un atajo (shorthand) con 1, 2 o 4 números. Contar los números te dice qué lado es cuál:

Se ve asíSe lee…
padding: 10px;Un valor → los 4 lados iguales.
padding: 10px 20px;Dos valores → arriba/abajo 10, izq/der 20. (vertical, luego horizontal)
padding: 10px 20px 30px 40px;Cuatro → como un reloj desde arriba, en sentido horario: arriba, derecha, abajo, izquierda.
padding: 10px 20px 30px;Tres → raro. arriba 10, izq/der 20, abajo 30. (reconócelo, no lo memorices)

Las filas con ● son las que verás el 80% de las veces. La misma lógica sirve idéntica para margin. Y si quieres un solo lado, lo verás con nombre: margin-top, padding-left… sin misterio.

El detalle que descoloca: box-sizing

Cuando leas width: 300px, hay una pregunta trampa: ¿300px es la caja entera, o solo el contenido (y padding + border se suman encima)? Lo decide una propiedad: 

box-sizing: border-box;   /* width = la caja ENTERA. padding y border caben dentro */
box-sizing: content-box;  /* (por defecto) width = solo el contenido; padding+border se SUMAN */

Núcleo del 80%: casi siempre es border-box

El código moderno suele poner * { box-sizing: border-box; } al principio (¡mira el course.css de este propio curso!). Significa: cuando veas width: 300px, la caja mide 300px en total, ya con su padding y borde incluidos. Es el comportamiento intuitivo — por eso todo el mundo lo activa.

Leer un border: tres piezas en fila

El borde casi siempre viene como un atajo de tres valores seguidos (y el orden entre ellos da igual): grosor · estilo · color. 

border: 1px solid navy;
/*        │    │     └─ color:  azul marino          */
/*        │    └─ estilo: solid = línea continua     */
/*        └─ grosor: 1px de grosor                   */

Los estilos que más verás: solid (línea maciza), dashed (guiones), none (sin borde). Con esas tres piezas lees el borde de casi cualquier web. La misma idea vale para border-top, border-bottom… si solo quieren un lado.

Compruébalo — recuerda sin releer

Pregunta 1
¿Qué propiedad separa un elemento de sus vecinos (espacio por fuera del borde)?

Correcto. margin es el espacio EXTERIOR: aleja la caja de las de al lado.
Por fuera del borde va el margin. El padding es por dentro (aleja el contenido de su propio borde); el border es la línea en medio.

Pregunta 2
¿Cómo se lee margin: 5px 30px; (dos valores)?

Correcto. Dos valores = vertical primero, horizontal después: arriba/abajo 5px, izquierda/derecha 30px.
Con dos valores el orden es vertical → horizontal: el primero (5px) es arriba/abajo, el segundo (30px) es izquierda/derecha.

Pregunta 3
Con box-sizing: border-box;, un elemento con width: 200px; y padding: 20px; ocupa en pantalla un ancho total de…

Correcto. Con border-box, width es la caja entera: el padding cabe DENTRO de esos 200px.
border-box hace que width sea el total: 200px ya incluyen el padding. (Con content-box sí serían 240px — por eso confunde tanto.)

Pregunta 4 · repaso
En .tarjeta p { padding: 8px; }, ¿a qué elementos se aplica ese padding? (de la Lección 02)

Correcto. El espacio = "dentro de": los p que estén dentro de un elemento con clase tarjeta. ¡La trampa del espacio, dominada!
Cuidado con el espacio: .tarjeta p = los p DENTRO de algo con clase tarjeta (no los p que tengan esa clase — eso sería p.tarjeta, pegado).

Tu misión de lectura (45 segundos)

Traduce esta regla real en voz alta, capa por capa, antes de seguir:

.boton {
  padding: 12px 24px;
  border: 1px solid navy;
  margin: 16px;
}

Pista: ¿cuánto aire hay dentro del botón, arriba/abajo vs. izquierda/derecha? ¿Y cuánto lo separa de sus vecinos?   (Respuesta: relleno interior de 12px arriba/abajo y 24px a los lados; un borde azul fino de 1px; y 16px de aire por fuera en los cuatro lados.)

📖 Fuente primaria recomendada

Lee «El modelo de caja» en MDN Web Docs (Mozilla, en español): el mismo tema con diagramas interactivos. Y lo más útil de todo para leer cajas: abre las DevTools del navegador (clic derecho → «Inspeccionar»), selecciona cualquier elemento y mira el diagrama de caja de colores en la pestaña de estilos — es exactamente estas cuatro capas, en vivo, sobre webs reales.
