Lección 01 · Leer CSS

La anatomía de una regla

Todo el CSS del mundo está hecho de una sola pieza repetida miles de veces: la regla. Si aprendes a "traducir" una regla, ya puedes empezar a leer cualquier hoja de estilos. Ese es el logro de hoy.

El patrón que se repite siempre

Mira este fragmento. No importa si no reconoces las palabras todavía — fíjate solo en la forma: 

h1 {
  color: red;
  font-size: 32px;
}

Esta regla se lee, en español llano:

«A los h1 ponles el color rojo y un tamaño de letra de 32 píxeles.»

Las cuatro partes

Cada regla tiene siempre estas piezas. Aprende sus nombres — los usaremos en todas las lecciones:

h1 { color: red; }
selector → a qué apunta   propiedad → qué cambia   valor → cómo queda

1 · Selector ■

Va antes de la llave {. Responde: ¿a qué elementos de la página afecta esta regla? Aquí, h1 significa "todos los títulos <h1> del HTML". Los selectores son un mundo entero — los veremos a fondo en la próxima lección.

2 · Bloque de declaraciones { … }

Todo lo que va entre llaves { }. Es la lista de cambios que se aplican al selector. Dentro puede haber una declaración o cien.

3 · Propiedad ■

El qué vas a cambiar: color, font-size, margin, background... Va a la izquierda de los dos puntos :. Hay cientos, pero unas 30 cubren la mayoría del código real.

4 · Valor ■

El cómo: el ajuste concreto de esa propiedad. Va a la derecha de los dos puntos : y termina en punto y coma ;.

La combinación clave

propiedad: valor; juntos se llaman una declaración. Leer CSS es, en el fondo, leer declaración tras declaración: «esta propiedad queda con este valor».

Señales de puntuación para leer rápido

: separa propiedad de valor (léelo como "queda en").   ; termina cada declaración.   { } agrupan lo que le pasa a un selector.   /* … */ es un comentario: texto ignorado por el navegador, notas para humanos.

Compruébalo tú — recuerda, no releas

Sin volver a mirar arriba, responde. Equivocarte aquí es parte del método: el esfuerzo de recordar es lo que fija el conocimiento.

Pregunta 1
En p { margin: 0; }, ¿cuál es el selector?

Correcto. p va antes de la llave: apunta a todos los párrafos.
p es el selector: va antes de { e indica a qué apunta. margin es la propiedad.

Pregunta 2
En a { color: blue; }, la pieza blue es el/la…

Correcto. blue va tras los dos puntos: es el cómo queda el color.
Fíjate en la posición: tras los : siempre va el valor. color es la propiedad.

Pregunta 3
¿Cómo se llama color: blue; completo, propiedad y valor juntos?

Correcto. propiedad: valor; es una declaración.
Es una declaración: la pareja propiedad: valor;. Varias declaraciones forman el bloque entre { }.

Tu misión de lectura (30 segundos)

Traduce esta regla en voz alta, parte por parte, antes de seguir:

/* tarjeta de la web */
.card {
  background: white;
  padding: 16px;
}

(Selector .card, dos declaraciones. El . delante significa "clase" — justo lo que veremos después. ¿Qué crees que hace ese punto? Guárdalo para la próxima lección.)

📖 Fuente primaria recomendada

Lee «Empezar con CSS» en MDN Web Docs (Mozilla, en español). Es la referencia más confiable de la web; la primera parte muestra reglas reales con su selector, propiedad y valor, justo lo de hoy, con ejemplos que puedes editar.
